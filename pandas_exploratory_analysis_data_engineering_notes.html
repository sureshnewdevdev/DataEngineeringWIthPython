Core Concept

2. What is Exploratory Analysis in Data Engineering?

Exploratory Data Analysis, or EDA, means examining source data to understand its structure, quality, distribution, and behavior. For data engineering, EDA is mainly used to prepare better pipeline rules.

Source Understanding

Find how many rows, columns, files, and tables exist.

Schema Profiling

Check column names, data types, date formats, and unexpected fields.

Quality Checks

Identify nulls, duplicates, invalid values, and outliers.

Business Patterns

Analyze trends by date, region, customer, product, or category.

Interview answer: Exploratory analysis in data engineering is the process of profiling source data using tools such as pandas to identify schema, quality, and business pattern issues before building production pipelines.

6. Schema Issues and Data Type Problems

Schema issues are common in source systems. Examples include wrong data types, inconsistent column names, missing columns, and date format issues.

Common Schema Issues

IssueExampleImpactAction
Wrong data typeAmount loaded as textAggregation failsConvert using pd.to_numeric()
Date format mismatch01/02/2026 vs 2026-02-01Wrong date interpretationUse pd.to_datetime()
Column name mismatchCustomer Id vs CustomerIDPipeline failsStandardize column names
Missing columnTaxAmount not availableTransformation failsAdd validation rule
Extra columnUnexpected source fieldSchema driftLog and review

Schema Validation Code

expected_columns = ["OrderId", "CustomerId", "OrderDate", "Amount", "City"]

actual_columns = df.columns.tolist()

missing_columns = [col for col in expected_columns if col not in actual_columns]
extra_columns = [col for col in actual_columns if col not in expected_columns]

print("Missing columns:", missing_columns)
print("Extra columns:", extra_columns)

Data Type Conversion

df["OrderDate"] = pd.to_datetime(df["OrderDate"], errors="coerce")
df["Amount"] = pd.to_numeric(df["Amount"], errors="coerce")

print(df.dtypes)
Interview answer: Schema profiling helps identify missing columns, unexpected columns, wrong data types, schema drift, and date parsing issues before production ingestion.

7. Data Quality Checks using pandas

Data quality checks help identify bad records before they enter Silver or Gold layers.

Null Check

null_summary = df.isnull().sum().reset_index()
null_summary.columns = ["Column", "NullCount"]

print(null_summary)

Duplicate Check

duplicate_count = df.duplicated().sum()
duplicate_order_count = df["OrderId"].duplicated().sum()

print("Duplicate rows:", duplicate_count)
print("Duplicate OrderId:", duplicate_order_count)

Invalid Value Check

invalid_amount = df[df["Amount"] <= 0]
invalid_dates = df[df["OrderDate"].isnull()]

print("Invalid amount records:", len(invalid_amount))
print("Invalid date records:", len(invalid_dates))

Reusable Quality Summary Function

def data_quality_summary(df):
    summary = {
        "total_rows": len(df),
        "total_columns": len(df.columns),
        "duplicate_rows": df.duplicated().sum(),
        "null_counts": df.isnull().sum().to_dict()
    }
    return summary

quality_report = data_quality_summary(df)
print(quality_report)
Best practice: Convert repeated checks into reusable functions and include them in the pipeline validation step.

Topic 6

9. Outlier and Anomaly Detection

Outliers are values that are unusually high, low, or different from normal records. They may be valid business cases or data quality issues.

Describe Numeric Data

print(df["Amount"].describe())

Find High-Value Transactions

high_value_orders = df[df["Amount"] > df["Amount"].quantile(0.95)]

print(high_value_orders)

IQR-based Outlier Check

q1 = df["Amount"].quantile(0.25)
q3 = df["Amount"].quantile(0.75)
iqr = q3 - q1

lower_limit = q1 - 1.5 * iqr
upper_limit = q3 + 1.5 * iqr

outliers = df[(df["Amount"] < lower_limit) | (df["Amount"] > upper_limit)]

print("Outlier count:", len(outliers))
Important: Do not remove outliers blindly. Confirm with business users whether they are valid transactions or data errors.

10. Automated Profiling Report with pandas

A profiling report gives a quick summary of schema, data types, nulls, duplicates, and key statistics.

def create_profile_report(df):
    profile = []

    for column in df.columns:
        profile.append({
            "column_name": column,
            "data_type": str(df[column].dtype),
            "null_count": df[column].isnull().sum(),
            "null_percentage": round(df[column].isnull().mean() * 100, 2),
            "unique_count": df[column].nunique(),
            "sample_value": df[column].dropna().iloc[0] if df[column].dropna().shape[0] > 0 else None
        })

    return pd.DataFrame(profile)

profile_df = create_profile_report(df)
print(profile_df)

Save Profiling Report

profile_df.to_csv("source_profile_report.csv", index=False)
Best practice: Save the profiling report and share it with business analysts, data architects, and source-system owners before finalizing pipeline rules.

11. Corporate Use Case: Sales Source-System EDA

Business Scenario

A retail company sends daily sales files. Before building the ingestion pipeline, the data engineer must explore the file to understand schema issues, nulls, duplicates, invalid amount values, date issues, and sales patterns.

Complete EDA Script

import pandas as pd

df = pd.read_csv("source_sales.csv")

print("Basic Information")
print("Rows and columns:", df.shape)
print("Columns:", df.columns.tolist())
print(df.dtypes)

print("Null Summary")
print(df.isnull().sum())

print("Duplicate Summary")
print("Duplicate rows:", df.duplicated().sum())
print("Duplicate OrderId:", df["OrderId"].duplicated().sum())

df["OrderDate"] = pd.to_datetime(df["OrderDate"], errors="coerce")
df["Amount"] = pd.to_numeric(df["Amount"], errors="coerce")

print("Invalid Records")
print("Invalid dates:", df["OrderDate"].isnull().sum())
print("Invalid amounts:", (df["Amount"] <= 0).sum())

print("Business Patterns")
print("Sales by city")
print(df.groupby("City")["Amount"].sum().sort_values(ascending=False))

print("Top customers")
print(df.groupby("CustomerId")["Amount"].sum().sort_values(ascending=False).head(10))
