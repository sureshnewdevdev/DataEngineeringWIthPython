🧱 Core services: S3, EC2, Regions & AZ

🪣 Amazon S3

  • Object storage — unlimited scale, 11 nines durability.
  • Key concepts: Buckets, objects, prefixes, versions.
  • Storage classes: Standard, IA, Glacier, Intelligent‑Tiering.
  • Use in data engineering: data lake, staging, backups, artifacts.
  • Best practice: enable versioning, lifecycle rules, encryption (SSE‑S3/KMS).
⚡ S3 event notifications → trigger pipelines

⚙️ Amazon EC2

  • Elastic Compute — scalable virtual servers (instances).
  • Instance families: General, compute‑optimized, memory‑optimized, storage‑optimized.
  • Data engineering workloads: ETL runners, notebooks (EMR on EC2), custom processing.
  • Key features: AMIs, instance profiles (IAM), EBS volumes, security groups.
  • Cost: On‑Demand, Reserved, Spot (great for fault‑tolerant ETL).
📦 EBS + instance store · ephemeral vs persistent

🌍 Regions

Geographical area (e.g., us‑east‑1, eu‑west‑2). Choose for data residency, latency, and service availability.

📡 Availability Zones (AZ)

Isolated data centers within a region. Multi‑AZ for high availability & fault tolerance. S3 (standard) is AZ‑redundant.

☁️ Cloud infrastructure: storage, compute, data movement

📦 Storage · Compute · Movement

  • Storage: S3 (data lake, raw/curated), EBS (block storage for EC2), EFS (shared file).
  • Compute: EC2 (custom ETL), EMR (managed Hadoop/Spark), Glue (serverless Spark), Lambda (event‑driven).
  • Data movement: AWS DMS (migration), Glue ETL, Kinesis (streaming), S3 Transfer Acceleration, VPC endpoints.
  • Orchestration: Step Functions, MWAA (Airflow), EventBridge.

🔁 DataOps principle: treat infrastructure as code, automate movement, monitor lineage.

🧩 DataOps thread · environment‑aware config, naming, docs

🌱 Environment‑aware configuration

  • Use parameter store (SSM) or Secrets Manager for env‑specific values.
  • Prefix variables: dev, staging, prod — e.g., /dataeng/dev/s3_bucket.
  • Leverage CloudFormation / Terraform with conditionals.
  • Never hardcode: use ${ENV} placeholders in scripts.

🏷️ Cloud resource naming

  • Pattern: {org}-{env}-{service}-{name}-{random}
  • Example: acme-dev-s3-raw-7f3a, acme-prod-ec2-spark-01
  • Tags: Environment, Project, CostCenter, Owner.
  • Automate tagging with IAM policies or CloudFormation.

📄 Documentation of deployment steps

  • Maintain a “runbook” in markdown (or Confluence) with prerequisites, commands, and rollback.
  • Include architecture diagram (e.g., Lucidchart) and data flow.
  • Use README in your infra repo: ./deploy.sh --env dev etc.
  • Capture dependency matrix (IAM, VPC, subnets, S3 buckets).

🤖 Generating scripts, checklists & architecture notes with Claude/Copilot

🧠 Prompt engineering for infra code: “Generate a bash script that creates an S3 bucket with versioning, sets a lifecycle policy, and attaches a bucket policy for read‑only access from a specific EC2 role. Use environment variables for bucket name and region.”
#!/bin/bash
# generate-s3-infra.sh · environment‑aware setup
# usage: ./generate-s3-infra.sh dev us‑east‑1

ENV="${1:-dev}"
REGION="${2:-us‑east‑1}"
BUCKET_NAME="acme-${ENV}-data-lake-$(date +%s)"

echo "🌐 Creating bucket in region $REGION for env $ENV"

# create bucket with versioning and encryption
aws s3api create-bucket --bucket $BUCKET_NAME --region $REGION --create-bucket-configuration LocationConstraint=$REGION
aws s3api put-bucket-versioning --bucket $BUCKET_NAME --versioning-configuration Status=Enabled
aws s3api put-bucket-encryption --bucket $BUCKET_NAME --server-side-encryption-configuration '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"AES256"}}]}'

# lifecycle: transition to Glacier after 90 days
aws s3api put-bucket-lifecycle-configuration --bucket $BUCKET_NAME --lifecycle-configuration '{"Rules":[{"Status":"Enabled","Prefix":"","Transitions":[{"Days":90,"StorageClass":"GLACIER"}]}]}'

echo "✅ S3 bucket $BUCKET_NAME ready · tags: env=$ENV"
aws s3api put-bucket-tagging --bucket $BUCKET_NAME --tagging '{"TagSet":[{"Key":"Environment","Value":"'$ENV'"},{"Key":"Project","Value":"data-lake"}]}'

📜 EC2 + IAM role script

# create-ec2-with-role.sh
ROLE_NAME="dataeng-ec2-${ENV}"
INSTANCE_PROFILE="dataeng-profile-${ENV}"
# create instance profile and attach policy
aws iam create-role --role-name $ROLE_NAME --assume-role-policy-document '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Principal":{"Service":"ec2.amazonaws.com"},"Action":"sts:AssumeRole"}]}'
aws iam attach-role-policy --role-name $ROLE_NAME --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess
aws iam create-instance-profile --instance-profile-name $INSTANCE_PROFILE
aws iam add-role-to-instance-profile --role-name $ROLE_NAME --instance-profile-name $INSTANCE_PROFILE
# launch EC2 (t3.medium, with user‑data for spark env)
aws ec2 run-instances --image-id ami-0abcdef1234567890 --instance-type t3.medium --iam-instance-profile Name=$INSTANCE_PROFILE --user-data "#!/bin/bash \n apt update && apt install -y openjdk-11"

🧭 Architecture notes & deployment blueprint

📐 Reference architecture: Data Lake + ETL (S3 + EC2 + Glue)

  • Ingestion: S3 (raw zone) ← Kinesis Firehose / DMS / SDK uploads.
  • Processing: EC2 (Spark cluster) or Glue job reading from S3, writing to curated zone.
  • Orchestration: Step Functions / Airflow triggers EC2 or Glue on S3 events.
  • Consumption: Athena / Redshift Spectrum / QuickSight.
  • Security: IAM roles, KMS, VPC endpoints, S3 bucket policies.

🧩 DataOps in action: CI/CD pipeline (CodeCommit+CodePipeline) that deploys CloudFormation stacks per environment.

🤖 Copilot / Claude prompt example: “Write a bash script that creates a full data‑eng stack: VPC (with 2 subnets), S3 bucket, EC2 instance with IAM role, and an S3 gateway endpoint. Use environment variables and include error handling.”
#!/bin/bash -e
# deploy-dataeng-stack.sh · uses env: ENV, REGION, VPC_CIDR
ENV="${ENV:-dev}"; REGION="${REGION:-us‑east‑1}"
VPC_CIDR="10.0.0.0/16"; SUBNET1_CIDR="10.0.1.0/24"; SUBNET2_CIDR="10.0.2.0/24"

echo "🚀 Deploying stack for $ENV in $REGION"
# create VPC (simplified)
VPC_ID=$(aws ec2 create-vpc --cidr-block $VPC_CIDR --query 'Vpc.VpcId' --output text)
aws ec2 create-subnet --vpc-id $VPC_ID --cidr-block $SUBNET1_CIDR --availability-zone ${REGION}a
aws ec2 create-subnet --vpc-id $VPC_ID --cidr-block $SUBNET2_CIDR --availability-zone ${REGION}b
# create S3 gateway endpoint (for private subnet)
aws ec2 create-vpc-endpoint --vpc-id $VPC_ID --service-name com.amazonaws.$REGION.s3 --route-table-ids $(aws ec2 describe-route-tables --filters "Name=vpc-id,Values=$VPC_ID" --query 'RouteTables[0].RouteTableId' --output text)
# S3 bucket (versioned, encrypted) with env tag
BUCKET="dataeng-${ENV}-${REGION}-$(date +%s)"
aws s3api create-bucket --bucket $BUCKET --region $REGION --create-bucket-configuration LocationConstraint=$REGION >/dev/null
aws s3api put-bucket-versioning --bucket $BUCKET --versioning-configuration Status=Enabled
aws s3api put-bucket-encryption --bucket $BUCKET --server-side-encryption-configuration '{"Rules":[{"ApplyServerSideEncryptionByDefault":{"SSEAlgorithm":"AES256"}}]}'
echo "✅ Stack ready · VPC: $VPC_ID · S3: $BUCKET"
# output for documentation
cat <<EOF > ./stack-output-${ENV}.txt
ENV: $ENV | REGION: $REGION
VPC_ID: $VPC_ID
S3_BUCKET: $BUCKET
SUBNETS: ${REGION}a, ${REGION}b
ENDPOINT: S3 gateway attached
# next steps: launch EC2 or Glue
EOF
