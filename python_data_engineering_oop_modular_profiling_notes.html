4. Reusable Functions

Reusable functions help avoid duplicate code. A function should ideally do one clear task.

Beginner Style

import pandas as pd

sales_df = pd.read_csv("sales.csv")
customer_df = pd.read_csv("customers.csv")
product_df = pd.read_csv("products.csv")

Reusable Function

import pandas as pd

def read_csv_file(file_path):
    """Read a CSV file and return a DataFrame."""
    return pd.read_csv(file_path)

sales_df = read_csv_file("sales.csv")
customer_df = read_csv_file("customers.csv")

Validation Function

def validate_required_columns(df, required_columns):
    missing_columns = [c for c in required_columns if c not in df.columns]
    if missing_columns:
        raise ValueError(f"Missing columns: {missing_columns}")
    return True
Best practice: Write small functions with clear names. Avoid one function doing too many responsibilities.

5. OOP Concepts for Data Engineering

OOP groups related data and behavior together. In data engineering, OOP is useful for readers, transformers, validators, and loaders.

OOP ConceptMeaningData Engineering Example
ClassBlueprint for objects.CsvReader, DataValidator
ObjectInstance of a class.reader = CsvReader("sales.csv")
EncapsulationKeep related data and methods together.Reader class has file path and read method.
InheritanceReuse common behavior.BaseReader parent, CsvReader child.
PolymorphismSame method name, different behavior.read() for CSV, JSON, API.
class CsvReader:
    def __init__(self, file_path):
        self.file_path = file_path

    def read(self):
        import pandas as pd
        return pd.read_csv(self.file_path)

reader = CsvReader("sales.csv")
sales_df = reader.read()
Trainer explanation: A class is useful when a function needs to work with state, configuration, or multiple related operations.

6. Pipeline Classes

A pipeline usually has extract, transform, validate, and load steps. We can represent this using a class.

import pandas as pd

class SalesPipeline:
    def __init__(self, source_path, target_path):
        self.source_path = source_path
        self.target_path = target_path
        self.df = None

    def extract(self):
        self.df = pd.read_csv(self.source_path)
        return self

    def transform(self):
        self.df = self.df.drop_duplicates()
        self.df = self.df[self.df["Amount"] > 0]
        self.df["TaxAmount"] = self.df["Amount"] * 0.18
        return self

    def load(self):
        self.df.to_csv(self.target_path, index=False)
        return self

pipeline = SalesPipeline("bronze_sales.csv", "silver_sales.csv")
pipeline.extract().transform().load()
Best practice: For small scripts, functions are enough. For larger reusable pipelines, classes make the design cleaner.

7. Modular Scripting

Modular scripting means splitting code into multiple files based on responsibility.

data_engineering_project/
├── config/
│   └── config.yaml
├── src/
│   ├── readers.py
│   ├── transformers.py
│   ├── validators.py
│   ├── loaders.py
│   └── pipeline.py
├── tests/
│   └── test_validators.py
├── logs/
└── main.py

readers.py

import pandas as pd

def read_csv(file_path):
    return pd.read_csv(file_path)

transformers.py

def clean_sales_data(df):
    df = df.drop_duplicates()
    df = df[df["Amount"] > 0]
    df["TaxAmount"] = df["Amount"] * 0.18
    return df

main.py

from src.readers import read_csv
from src.transformers import clean_sales_data

df = read_csv("bronze_sales.csv")
clean_df = clean_sales_data(df)
clean_df.to_csv("silver_sales.csv", index=False)
Interview answer: Modular scripting separates code into logical modules such as readers, transformers, validators, loaders, and pipeline runners.

8. Configuration and Logging

Production pipelines should not hardcode file paths, database names, or environment settings.

config.yaml

source_path: "bronze_sales.csv"
target_path: "silver_sales.csv"
minimum_amount: 0
tax_percentage: 18

Read Configuration

import yaml

def load_config(config_path):
    with open(config_path, "r") as file:
        return yaml.safe_load(file)

config = load_config("config/config.yaml")

Logging Example

import logging

logging.basicConfig(
    filename="logs/pipeline.log",
    level=logging.INFO,
    format="%(asctime)s - %(levelname)s - %(message)s"
)

logging.info("Pipeline started")

try:
    logging.info("Pipeline completed successfully")
except Exception as ex:
    logging.error(f"Pipeline failed: {ex}")
    raise
Best practice: Use configuration for values that change between development, test, and production. Use logging to understand execution.

9. Profiling Concepts

Profiling means measuring code performance to find slow parts, memory-heavy operations, and bottlenecks.

Profiling AreaQuestionTool / Method
Execution TimeWhich step is slow?time, timeit, cProfile
Memory UsageWhich operation uses high memory?memory_profiler, pandas info()
Data VolumeHow many records are processed?len(df), count()
I/O TimeIs reading/writing slow?Log start and end time

Simple Time Profiling

import time
import pandas as pd

start_time = time.time()

df = pd.read_csv("large_sales.csv")
df = df[df["Amount"] > 0]

end_time = time.time()
print("Execution time:", end_time - start_time, "seconds")

Profiling Decorator

import time

def profile_time(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} took {end - start:.2f} seconds")
        return result
    return wrapper

@profile_time
def transform_sales(df):
    return df[df["Amount"] > 0]
Trainer explanation: Do not optimize blindly. First measure, then optimize the actual slow part.

10. Data Quality with Reusable Validators

Data quality checks should be reusable and fail early when important rules are broken.

class DataValidator:
    def __init__(self, df):
        self.df = df
        self.errors = []

    def check_not_null(self, column):
        null_count = self.df[column].isnull().sum()
        if null_count > 0:
            self.errors.append(f"{column} has {null_count} null values")
        return self

    def check_positive(self, column):
        invalid_count = (self.df[column] <= 0).sum()
        if invalid_count > 0:
            self.errors.append(f"{column} has {invalid_count} invalid values")
        return self

    def validate(self):
        if self.errors:
            raise ValueError(self.errors)
        return True

validator = DataValidator(sales_df)
validator.check_not_null("OrderId").check_positive("Amount").validate()
Best practice: Keep validations close to the pipeline and fail early when critical data quality rules are broken.

11. Real-World Use Case: Modular Sales ETL Pipeline

Business Scenario

A retail company receives daily sales files. The Data Engineering team must build a reusable Python pipeline to read the sales file, clean invalid records, validate required columns, add tax amount, and write the cleaned file for analytics.

import pandas as pd
import logging
import time

class SalesETLPipeline:
    def __init__(self, source_path, target_path):
        self.source_path = source_path
        self.target_path = target_path
        self.df = None

    def extract(self):
        logging.info("Reading source file")
        self.df = pd.read_csv(self.source_path)
        return self

    def validate(self):
        required_columns = ["OrderId", "CustomerId", "Amount"]
        for column in required_columns:
            if column not in self.df.columns:
                raise ValueError(f"Missing required column: {column}")
        return self

    def transform(self):
        self.df = self.df.drop_duplicates(subset=["OrderId"])
        self.df = self.df[self.df["Amount"] > 0]
        self.df["TaxAmount"] = self.df["Amount"] * 0.18
        return self

    def load(self):
        self.df.to_csv(self.target_path, index=False)
        return self

    def run(self):
        start = time.time()
        self.extract().validate().transform().load()
        end = time.time()
        logging.info(f"Pipeline completed in {end - start:.2f} seconds")

logging.basicConfig(level=logging.INFO)

pipeline = SalesETLPipeline("bronze_sales.csv", "silver_sales.csv")
pipeline.run()
